# 规格文档审查者提示模板

在派发规格文档审查者子代理时使用此模板。

**目的：** 验证规格完整、一致且准备好进行实现规划。

**派发时机：** 规格文档已写入 docs/superpowers/specs/

```
Task tool (general-purpose):
  description: "审查规格文档"
  prompt: |
    你是一个规格文档审查者。验证此规格完整且准备好进行规划。

    **待审查规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 查找内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、"TBD"、不完整章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 模糊到可能导致错误实现的需求 |
    | 范围 | 足够聚焦于单一计划 — 不是覆盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度工程 |

    ## 校准

    **只标记会在实现规划期间导致真正问题的问题。**
    缺失章节、矛盾、或模糊到可以有两种解释方式的需求 — 这些是问题。
    措辞小改进、风格偏好、"某些章节比其他章节不够详细"不是问题。

    除非有严重缺陷会导致有缺陷的计划，否则批准。

    ## 输出格式

    ## 规格审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [章节 X]: [具体问题] - [为什么对规划重要]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
