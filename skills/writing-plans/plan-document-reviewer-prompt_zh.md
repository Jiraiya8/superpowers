# 计划文档审查者提示模板

在分派计划文档审查子代理时使用此模板。

**目的：** 验证计划是否完整、是否符合规范、任务分解是否合理。

**分派时机：** 完整计划编写完成后。

```
Task tool (general-purpose):
  description: "审查计划文档"
  prompt: |
    你是一个计划文档审查者。验证此计划完整且准备好进行实现。

    **待审查计划：** [PLAN_FILE_PATH]
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 查找内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、不完整任务、缺失步骤 |
    | 规格一致性 | 计划覆盖规格需求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可操作 |
    | 可构建性 | 工程师能否按此计划工作而不被卡住？ |

    ## 校准

    **只标记会在实现期间导致真正问题的问题。**
    实现者构建错误的东西或被卡住是一个问题。
    措辞小改进、风格偏好和"锦上添花"建议不是问题。

    除非有严重缺口 — 缺失规格需求、矛盾步骤、占位符内容或任务太模糊无法执行 — 否则批准。

    ## 输出格式

    ## 计划审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [Task X, Step Y]: [具体问题] - [为什么对实现重要]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
